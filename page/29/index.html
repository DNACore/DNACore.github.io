Xcoed运行时警告

发表于 2018-09-27 更新于 2019-05-09

Xcode 运行时警告

  • Class AAA is implemented in both BBB and CCC. One of the two will be used. Which one is undefined.
    原因：同一个类的实现有两个，系统只会选择一个。实现分别在 BBB 和 CCC 里面。
    解决：找到这两个实现，删除其中一个即可。更多的情况是，一份实现是以 .m 文件存在的，另一份实现是以静态库存在的。
    参考： https://stackoverflow.com/questions/30581884/class-is-implemented-in-both-one-of-the-two-will-be-used

dyld: Library not loaded

问题描述：
@rpath/Charts.framework/Charts
Referenced from: /var/containers/Bundle/Application/FEFF646B-5902-4015-B159-C141EB0E8DC0/test.app/test
Reason: image not found
解决：
在 TARGETS —> General —> Embedded Binaries 下，点击 + 按钮，选择 Charts.framework就可以解决问题

  • A valid provisioning profile for this executable was not found for debug mode
    看看设备的时间是否正确。
    参考：https://stackoverflow.com/questions/4167466/a-valid-provisioning-profile-for-this-executable-was-not-found-for-debug-mode

  • 问题 Error Domain=WebKitErrorDomain Code=101 "The URL can’t be shown"
    问题描述：

    Printing description of error:
    Error Domain=WebKitErrorDomain Code=101 “The URL can’t be shown” UserInfo={NSErrorFailingURLStringKey=hhrdkscheme://showTitleBar?show=1, NSLocalizedDescription=The URL can’t be shown, NSErrorFailingURLKey=hhrdkscheme://showTitleBar?show=1}

问题原因：有不能加载的url，比如说协议不正确。例如：加载了app的自定义协议。
解决办法：处理不能加载的协议shouldStartLoadWithRequest:返回个NO.
解决结果：问题解决。

通过 Runtime 添加属性

一般情况下，类的 category 添加了一个 property 之后，只会生成对应的 get 和 set 方法声明，并不会生成这两个方法的实现，也不会生成对应的带下划线的方法。
用 Runtime 添加属性：

方法说明

objc_setAssociatedObject(id object, const void *key, id value, objc_AssociationPolicy policy)

  • id object
    添加属性的目标对象，传入需要添加属性的实例。

  • const void *key
    一个指针，传入地址，

  • id value
    传入属性的值

  • objc_AssociationPolicy policy
    关联策略，此处是个枚举值，与 @property 的修饰符有对应关系

id objc_getAssociatedObject(id object, const void *key)

获取某个关联属性的值

pod setup (根据测试，此步骤可以省略)
执行安装命令后会向~/.cocoapods/repos这里拉取git内容，大概有341MB。等待一段时间之后，如果没有错误的话 CocoaPods 就安装成功了。

为什么要执行pod setup？
所有的项目的 Podspec 文件都托管在 https://github.com/CocoaPods/Specs 第一次执行 pod setup 时，CocoaPods 会将这些podspec索引文件更新到本地的~/.cocoapods/目录下，这个索引文件比较大，有 300多M 左右，在执行一次pod install之后会增加到700多M。所以第一次更新时非常慢。

配置文件的书写

  1. 在 *.xcodeproj 所在的文件夹下建立 Podfile 为名，的无扩展名文本文件，文件需要的编码是 UTF-8 换行符为 UNIX 格式。
    这里根据文档，也可以执行 pod init 就会在当前目录生成一个 Podfile 文件。
  2. Podfile 的文件内容
    • platform :ios, '7.0' 最低支持的 iOS 版本。
    • target 'CloudStorageHelper' 这里是将要向哪个 Target 添加库文件。
    • pod 'AFNetworking', '~> 3.1.0' 需要添加的第三方依赖库和版本。
  • 只在Debug模式下加入pod库

  1. 删除依赖库对应的版本代码
  2. 执行 pod install 不确定 pod update 是否有同样的效果

iOS 创建推送证书教程

发表于 2017-03-14

iOS 创建推送证书教程

此教程帮助大家创建iOS设备在开发测试环境和生产环境使用的推送证书。

准备

每次我们向苹果申请证书，都需要生成一个证书请求文件。通过这个文件来获取我们需要的证书。

创建证书请求文件

  1. 打开钥匙串
  2. 在菜单栏的 钥匙串访问 中选中 证书助理 –> 从证书颁发机构请求证书…
  3. 在弹出的窗口中，填写好你的电子邮件地址，并且一定要选中 存储到磁盘 。我们稍后会用到生成的请求文件。
  4. 点击继续，提示我们选择一个存储位置，这里我们选择存储到桌面上，点存储。
  5. 提示创建成功，我们可以关闭窗口了。

创建 APS 证书

  1. 打开开发者网站的证书页面
    https://developer.apple.com/account/ios/certificate/
  2. 点击右上方的 ＋ 号开始创建证书。
  3. 在新页面中，有两种类型的推送证书可供我们选择。这里我们选择前面一种，也就是测试环境下的证书。
    1. Apple Push Notifiction service SSL (Sandbox) 测试用推送证书
    2. Apple Push Notifiction service SSL (Sandbox & Production) 生产环境证书
  4. 点击 Continue 进入下一个页面，要我们选择需要针对哪个 app 来创建推送证书，推送证书不支持通用 Bundle ID。选择好你对用的 Bundle ID 点击 Continue。
  5. 这里提示我们要创建一个 CSR 文件。我们上面意境创建过了，直接点击 Continue。
  6. 点击 chose file... 选择在上面一部分我们保存好的证书请求文件。之后点击 Continue。
  7. 至此，推送证书就创建完成，点击 Download 下载我们创建好的证书，点击 Done 完成。双击下载的证书即可自动导入钥匙串中。

动态改变导航栏的颜色和透明度

有一些应用在我们推拉滑动列表的时候会发现导航栏随着滑动的动作改变颜色和透明度。
这种效果其实实现起来是比较简单的。
这里利用的是 UIScrollview 的代理。每当 UIScrollView 滑动的时候就会掉用- (void)scrollViewDidScroll:(UIScrollView *)scrollView这个方法。我们只要在这个代理里面获取到 UIScrollView 的偏移量，根据偏移量来设置导航栏的颜色和透明度。
