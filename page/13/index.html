    MacBook73:~/Documents/ProjectPersonal/SKeyUtils/Example$ pod install
    Analyzing dependencies
    Fetching podspec for SKeyUtils from ../
    [!] CocoaPods could not find compatible versions for pod “SKeyUtils”:
    In Podfile:

    SKeyUtils (from `../`)

问题解决：

Shell-ln文件和目录链接命令

发表于 2019-03-13 更新于 2019-08-23
  • 软链接和硬连接的异同
    软链接又叫符号链接，使用ln命令的时候加上-s参数，创建的链接即为软链接。不带-s参数创建的链接为硬连接。
    实际使用情况发现了两者有一些不同之处：
  1. 我们有一个可执行文件PooFoo，这个可执行文件依赖当前目录下一些其他文件。当使用软链接把这个可执行文件链接到/usr/local/bin/目录的时候，直接执行命令没有问题。可执行文件可以正常找到当前目录下的其他依赖文件。可是当用硬连接来链接文件的时候，可执行文件便找不到依赖了。在可执行文件中打印了一下当前的执行环境，发现执行目录变为/usr/local/bin/，自然是找不到依赖了。这个时候再改成软链接，马上就可以正常使用了。
    由此，软链接更像是快捷方式。

  • 创建软链接
    创建软链接的话，在源文件被赋予可执行权限之前和之后创建的软链接的权限是稍有不同的。

获取GitHub中文件的直接下载连接

发表于 2019-03-07 更新于 2019-09-23

  • 直接在GitHub网页中打开一个文件，选择RAW查看，此时地址栏中的连接即为这个文件的直接链接。如果是二进制文件的话会有个DOWNLOAD按钮，是一样的。

  • 可以直接用鼠标指向Raw按钮，复制这个按钮指向的连接。

  • 针对图片，在网页打开图片页面之后，右键复制图片链接也是可以的。

  • Release 中的文件直接下载链接鼠标指上去右键复制链接就可以。

Finder 访达快捷键

  • 在新窗口中打开上一级文件夹 ⌘+ control + 上
  • 在当前窗口中打开上一级文件夹 ⌘ + option + 上
  • 打开用户文件夹⌘+Shift+H
  • 快速文件夹跳转⌘+Shift+G
  • 快速打开实用工具文件夹⌘+Shift+U
  • command+左击 Dock 栏程序：打开对应程序所在的位置。
  • 批量查看选中文件信息：⌘ + option +i
  • 显示隐藏文件：⌘+shift+.
  • 显示路径：cmd+option+p
  • 复制路径：cmd+option+c如果被复制的路径包含软链接，那么复制出来的路径是原始路径。
    另外的方法
    显示：defaults write com.apple.finder AppleShowAllFiles -bool true
    隐藏：defaults write com.apple.finder AppleShowAllFiles -bool false
