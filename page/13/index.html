获取GitHub中文件的直接下载连接

发表于 2019-03-07 更新于 2019-09-23

  • 直接在GitHub网页中打开一个文件，选择RAW查看，此时地址栏中的连接即为这个文件的直接链接。如果是二进制文件的话会有个DOWNLOAD按钮，是一样的。

  • 可以直接用鼠标指向Raw按钮，复制这个按钮指向的连接。

  • 针对图片，在网页打开图片页面之后，右键复制图片链接也是可以的。

  • Release 中的文件直接下载链接鼠标指上去右键复制链接就可以。

macOS系统

苹果官方网站提供的快捷键信息
https://support.apple.com/zh-cn/HT201236

Finder 访达快捷键

  • 在新窗口中打开上一级文件夹 ⌘+ control + 上
  • 在当前窗口中打开上一级文件夹 ⌘ + option + 上
  • 打开用户文件夹⌘+Shift+H
  • 快速文件夹跳转⌘+Shift+G
  • 快速打开实用工具文件夹⌘+Shift+U
  • command+左击 Dock 栏程序：打开对应程序所在的位置。
  • 批量查看选中文件信息：⌘ + option +i
  • 显示隐藏文件：⌘+shift+.
  • 显示路径：cmd+option+p
  • 复制路径：cmd+option+c如果被复制的路径包含软链接，那么复制出来的路径是原始路径。
    另外的方法
    显示：defaults write com.apple.finder AppleShowAllFiles -bool true
    隐藏：defaults write com.apple.finder AppleShowAllFiles -bool false

Windows 错误报告文件夹位置

也可以在注册表HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer
VolumeCaches查询到。
用系统自带的磁盘清理工具清理C盘的时候可以看到一堆Windows错误报告文件，这些文件的具体位置是：

  • 系统错误内存转储文件

  • Windows Error Reporting Archive Files
    DISM++里面的垃圾清理功能有错误报告路径
    用户存档的Windows错误报告文件
    %USERPROFILE%\AppData\Local\Microsoft\Windows\WER\ReportArchive

  • Windows Error Reporting Queue Files
    用户队列中的Windows错误报告文件
    %USERPROFILE%\AppData\Local\Microsoft\Windows\WER\ReportQueue

  • Windows Error Reporting System Archive Files
    系统存档的Windows错误报告文件
    %ALLUSERSPROFILE%\Microsoft\Windows\WER\ReportArchive

  • Windows Error Reporting System Queue Files
    系统队列中的Windows错误报告文件
    %ALLUSERSPROFILE%\Microsoft\Windows\WER\ReportQueue

  • Windows Upgrade Log Files
    Windows\\Panther
