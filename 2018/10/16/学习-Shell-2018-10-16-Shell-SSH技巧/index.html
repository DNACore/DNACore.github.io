config 配置文件

ssh 同一个git域名使用多个证书

说明：配置文件里面Host指的是git托管网站域名的别名，因为我们可能在一台电脑上有多个同一个域名下的git账户，这个时候就要用Host来区分这些账户。
相对应的，git仓库里面配置的git地址也要用别名，当对远程git仓库进行操作的时候，git程序会查找config配置文件里面别名对应的host。
举个例子，一下全部是使用ssh来管理git的情况：
默认情况下，我们的git地址是类似于git@github.com:AirIO/AirIO.github.io.git这样的地址格式，解析的时候它会从github.com域名下查找路径为AirIO/AirIO.github.io.git的仓库，然后使用SSH配置的密钥来操作仓库。
但是SSH针对一个域名只能绑定一个密钥对。如果我在github上又有另外一个账号MyGitHub2该怎么办？github上的密钥对已经配置成AirIO的了。
这个时候config文件就有个别名的概念，我们可以按照如下规则来配置config文件：

相对应的，我们git仓库的配置文件(注意，不是SSH的config配置文件)里面的git仓库地址也要小小做一下修改：
git@github.com:AirIO/AirIO.github.io.git
改为
git@GitHubAccount_1:AirIO/AirIO.github.io.git

git@github.com:MyGitHub2/MyGitHub2.git
改为
git@GitHubAccount_2:MyGitHub2/MyGitHub2.git
这样修改之后，当git再更新远程仓库的时候，会查找每个账户对应的SSH密钥。最终访问的域名还是github.com

当执行ssh -T git@GitHub_AirIO的时候，会从config文件里面查找Host为GitHub_AirIO的配置，而不是去查找Host为git@GitHub_AirIO的配置，因为git是用户名（默认情况下用户名都为git而不是git网站上自己账户的用户名）,GitHub_AirIO才是 Host。这一点一定要注意.
