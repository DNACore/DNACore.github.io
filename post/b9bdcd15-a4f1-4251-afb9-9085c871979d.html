Windows-将 Windows 启动至审核模式或 OOBE

将 Windows 启动至审核模式或 OOBE

可以使用审核模式来自定义计算机、添加应用程序和设备驱动程序，以及在 Windows 环境中测试计算机。

  • 启动至审核模式会在内置管理员帐户中启动计算机，该帐户随后在 generalize 配置阶段会自动删除。
  • 将计算机配置为启动至审核模式后，在将计算机配置为启动至开箱即用体验 (OOBE) 之前，该计算机默认会继续启动至审核模式。

备注

如果在审核模式下启动受密码保护的屏幕保护程序，将无法重新登录到系统。 登录后，系统将立即禁用用于在审核模式下登录的内置管理员帐户。 若要禁用屏幕保护程序，请通过 Windows 控制面板更改电源计划，或配置并部署自定义计划。 有关详细信息，请参阅创建自定义电源计划。

在全新安装时自动启动至审核模式

  • 若要将 Windows 配置为启动至审核模式，请添加 Microsoft-Windows-Deployment | Reseal | Mode = audit 应答文件设置。

    当 Windows 完成安装过程时，计算机将自动启动进入审核模式，并出现系统准备 (Sysprep) 工具。 有关在审核模式下使用 Sysprep 工具的详细信息，请参阅 Sysprep（通用化）Windows 安装。

备注

oobeSystem 配置阶段的应答文件中的设置不会在审核模式下显示。 有关启动至审核模式或 OOBE 时处理哪些应答文件设置的详细信息，请参阅配置阶段的工作原理。

手动启动至审核模式（在新安装或现有安装上）

  • 在 OOBE 屏幕上，按 CTRL+SHIFT+F3。

    Windows 将计算机重新启动至审核模式，并出现系统准备 (Sysprep) 工具。

备注

CTRL+SHIFT+F3 快捷键不会绕过 OOBE 过程的所有步骤，例如不会绕过 oobeSystem 配置阶段中运行脚本和应用应答文件设置的步骤。

在全新安装时自动启动至 OOBE

  • 若要将 Windows 配置为启动至 OOBE，请添加 Microsoft-Windows-Deployment | Reseal | Mode = oobe 应答文件设置。

    如果已将 Windows 映像配置为启动至 OOBE，但需要在审核模式下对映像进一步进行配置，请参阅修改已配置为启动至 OOBE 的现有映像。
